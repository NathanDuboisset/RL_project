\begin{abstract}
We investigate reinforcement learning approaches to the puzzle game \emph{Block Blast}, played on an $8\times8$ grid. The agent must place Tetromino-like pieces to clear full rows and columns. We study two environment variants: a single-piece variant (1P) where one piece is given at a time, and a three-piece variant (3P) where three pieces are presented simultaneously and must all be placed in a freely chosen order. We experiment with classical Deep Q-Network approaches (DDQN, Rainbow DQN) and introduce a \emph{Deep Value Network} (DVN) that operates on \emph{afterstates}---the board states resulting from each possible placement---instead of learning a standard Q-function. The DVN achieves significantly better performance than naive baselines with only 42\,461 parameters, compared to the $\sim$2.67M parameters needed by DQN-based models.
\end{abstract}

%=============================================================================
\section{Introduction}

Block Blast is a single-player puzzle game played on an $8\times8$ grid. At each turn, the player receives one or more Tetromino-like pieces and must place them on the grid. When a full row or column is formed, it is cleared and points are scored. The game ends when no more pieces can be placed.

This game presents several classic challenges for reinforcement learning:
\begin{itemize}
    \item \textbf{Long-horizon planning}: clearing lines requires setting up the board over many turns.
    \item \textbf{Large action space}: up to 64 positions per piece in the 1P variant, 192 in the 3P variant.
    \item \textbf{Sparse rewards}: most individual placements yield no line clears.
    \item \textbf{Combinatorial structure}: in the 3P variant, the order of the three pieces matters.
\end{itemize}

We organised our experiments in three phases. First, we designed the environment and iterated on the reward function. Second, we trained standard DQN-based agents (DDQN and Rainbow DQN). Third, we developed a Deep Value Network (DVN) operating on afterstates.

%=============================================================================
\section{Environment}

\subsection{Game Rules}

The game is played on an $8\times8$ binary grid (0 = empty, 1 = filled). Seven piece shapes are used, each randomly rotated by a multiple of 90° at the start of each episode:

\begin{center}
\begin{tabular}{ll}
\toprule
Shape & Description \\
\midrule
Dot        & $1\times1$ single cell \\
Square     & $2\times2$ block \\
Big Square & $3\times3$ block \\
Line-3     & $1\times3$ horizontal strip \\
Line-4     & $1\times4$ horizontal strip \\
L-small    & $2\times2$ L-shape \\
L-large    & $3\times3$ L-shape \\
\bottomrule
\end{tabular}
\end{center}

After each placement, every row and column that is entirely filled is cleared simultaneously. The agent must avoid reaching a state where no valid placement exists, as this ends the episode.

\subsection{Single-Piece Variant (1P)}

In the 1P variant, one piece is drawn uniformly at random at the start of each step. The observation space is a dictionary containing:
\begin{itemize}
    \item \texttt{board}: the $8\times8$ integer grid,
    \item \texttt{piece}: the piece shape encoded in a $5\times5$ padded grid,
    \item \texttt{valid\_placements}: an $8\times8$ binary mask of valid top-left positions,
    \item \texttt{placements\_result}: a precomputed $(8,8,8,8)$ tensor giving the resulting board for every candidate placement, alongside an $(8,8)$ reward map.
\end{itemize}

The action space is $\text{Discrete}(64)$, where action $a = 8r + c$ places the piece with its top-left corner at row $r$, column $c$.

\subsection{Three-Piece Variant (3P)}

In the 3P variant, three pieces are drawn at the start of each \emph{round}. The agent must place all three in a freely chosen order before a new round begins. The action space becomes $\text{Discrete}(192)$:
\[
a = \text{piece\_idx} \times 64 + r \times 8 + c
\]
A binary vector \texttt{pieces\_used} tracks which pieces have been played. A \texttt{combo} counter accumulates cleared lines and is reset when a full round completes without any clearing.

The environment also exposes \texttt{get\_t\_plus\_3\_candidates($\gamma$)}, which enumerates all $3!$ permutations and their valid positions, returning the terminal board, cumulative discounted reward, and action sequence for each candidate. This is used by the DVN 3P planner (Section~\ref{sec:dvn}).

\subsection{Reward Shaping}
\label{sec:reward}

\subsubsection{Initial Quadratic Reward (Abandoned)}

In early experiments, we used a quadratic bonus for clearing multiple lines at once:
\[
r = b \cdot n_{\text{cleared}}^2, \qquad b = 10
\]
The intuition was to strongly incentivise combos. However, clearing multiple lines simultaneously is rare early in training, making the reward signal extremely sparse. The agent received near-zero reward for the vast majority of steps, causing training to stall.

\subsubsection{Final Reward Function (1P)}

The final 1P reward is computed for each hypothetical afterstate:
\begin{equation}
r = \underbrace{r_{\text{survive}}}_{5.0}
  + \underbrace{b \cdot n_{\text{cleared}}}_{10 \times \#\text{lines}}
  + \underbrace{b \cdot \rho}_{\text{emptiness}}
  - \underbrace{p_{\text{no3x3}}}_{20 \text{ if no free }3\times3}
\end{equation}
where $\rho \in [0,1]$ is the fraction of empty cells (encourages keeping the board clean), and $p_{\text{no3x3}} = 20$ if no $3\times3$ window of empty cells exists after placement. Invalid moves incur a penalty of $-100$ and terminate the episode.

\subsubsection{Final Reward Function (3P)}

The 3P variant uses a combo-based reward:
\[
r = \begin{cases}
0.1 & \text{if no lines cleared} \\
b \cdot (n_{\text{cleared}}^2 + \text{combo}) & \text{if } n_{\text{cleared}} > 0
\end{cases}
\]
The \texttt{combo} counter accumulates across placements, encouraging sustained clearing streaks.

\section{DQN-Based Approaches}
\label{sec:dqn}

\subsection{Architecture}

Both DDQN and Rainbow share the same convolutional backbone, \texttt{BlockBlastCNNNet1P}:
\begin{enumerate}
    \item \textbf{Board branch}: two channels (board + valid mask), $8\times8$ \\ $\to$ Conv2d($2\to32$, $k=3$) $\to$ Conv2d($32\to64$, $k=3$) $\to$ Flatten $\to$ 4096 features.
    \item \textbf{Piece branch}: one channel ($5\times5$) $\to$ Conv2d($1\to32$, $k=3$) $\to$ Flatten $\to$ 800 features.
    \item \textbf{FC head}: $4896 \to 512 \to 256 \to 64$ Q-values.
\end{enumerate}
Total parameters: \textbf{2\,674\,464}.

\subsection{Double DQN (DDQN)}

Double DQN decouples action selection from action evaluation in the target:
\[
y_t = r_t + \gamma \cdot Q_{\theta^-}\!\left(s_{t+1},\; \arg\max_a Q_\theta(s_{t+1}, a)\right)
\]
where $\theta$ is the policy network and $\theta^-$ the target network (updated every 10 episodes). Invalid actions are masked with $-10^9$ before the argmax.

Training: $\varepsilon$ decayed $1.0\to0.05$ ($\times0.9995$/episode), replay buffer 10\,000, batch size 64, $\gamma=0.99$, Adam $\text{lr}=10^{-4}$.

\subsection{Rainbow DQN}

Rainbow combines five extensions:
\begin{itemize}
    \item \textbf{Distributional RL (C51)}: outputs a probability distribution over 51 atoms in $[{-10}, 10]$; loss is KL divergence.
    \item \textbf{Prioritized Experience Replay (PER)}: priority $\propto |\delta|^\alpha$ ($\alpha=0.5$), IS correction $\beta=0.4$.
    \item \textbf{N-step returns}: 3-step bootstrapped targets.
    \item \textbf{Noisy Networks}: NoisyLinear layers ($\sigma_0=0.5$) replace $\varepsilon$-greedy.
    \item \textbf{Dueling architecture}: separate value and advantage streams, $Q = V + A - \bar{A}$.
\end{itemize}

\subsection{Limitations}

Despite 1\,000 training episodes, neither DDQN nor Rainbow achieved consistent long-game performance:
\begin{enumerate}
    \item \textbf{Credit assignment}: a good placement may not produce a line clear for several steps; discounted credit is weak.
    \item \textbf{Q-function instability}: 64 outputs with a highly variable valid mask at each step.
    \item \textbf{Sample efficiency}: 2.67M parameters trained on only 1\,000 episodes is severely underfit.
    \item \textbf{Perception overhead}: the network must simultaneously learn piece recognition, valid placements, \emph{and} long-term board value.
\end{enumerate}

\section{Deep Value Network (DVN)}
\label{sec:dvn}

\subsection{Afterstate Formulation}

Instead of learning $Q(s, a)$, the DVN learns a \emph{state value function} $V_\theta(s')$ where $s'$ is the \emph{afterstate}: the board after placement and line clearing. Since the environment precomputes \texttt{placements\_result}, the agent enumerates all valid afterstates and selects:
\[
a^* = \arg\max_a \left[ r_a + \gamma \cdot V_\theta(s'_a) \right]
\]
Advantages over Q-learning: the network only evaluates boards (not board--action pairs), valid constraints are handled implicitly, and the value function captures board quality independently of the current piece.

\subsection{Architecture: Multi-Kernel CNN}

The value network \texttt{BlockBlastValueNet1PmultikernelFlattenned} uses a bank of eight parallel convolutional branches:

\begin{center}
\begin{tabular}{lcccc}
\toprule
Branch & Kernel & Filters & Wall padding & Output spatial \\
\midrule
Square $1\times1$ & $(1,1)$ & 1  & 0 & $8\times8$ \\
Square $2\times2$ & $(2,2)$ & 8  & 1 & $9\times9$ \\
Square $3\times3$ & $(3,3)$ & 16 & 1 & $8\times8$ \\
Square $4\times4$ & $(4,4)$ & 16 & 2 & $9\times9$ \\
Square $5\times5$ & $(5,5)$ & 16 & 2 & $8\times8$ \\
Square $8\times8$ & $(8,8)$ & 64 & 0 & $1\times1$ \\
Horizontal $1\times8$ & $(1,8)$ & 4  & 0 & $8\times1$ \\
Vertical $8\times1$   & $(8,1)$ & 4  & 0 & $1\times8$ \\
\bottomrule
\end{tabular}
\end{center}

Padding uses \texttt{ConstantPad2d} with value \textbf{1.0} so the kernel sees filled wall cells outside the board boundary. Line kernels ($1\times8$, $8\times1$) use no padding.

Each branch: Flatten $\to$ Linear$(\cdot\to16)$ $\to$ LeakyReLU $\to$ Linear$(16\to8)$.
All eight outputs are concatenated into a \textbf{64-dimensional} feature vector.

The role of each branch family:
\begin{itemize}
    \item $1\times1$: per-cell occupancy.
    \item $2\times2$--$5\times5$: local block patterns matching common piece shapes.
    \item $8\times8$: global board summary.
    \item $1\times8$, $8\times1$: full-row and full-column detectors---crucial for identifying nearly-complete lines.
\end{itemize}

\subsubsection{MLP Head}

\[
\text{Linear}(64 \to 64) \to \text{LeakyReLU} \to \text{Linear}(64 \to 16) \to \text{LeakyReLU} \to \text{Linear}(16 \to 1)
\]

Total parameters: \textbf{42\,461} ($63\times$ fewer than DDQN/Rainbow).

\subsection{Training}

The DVN is trained via temporal difference learning on afterstates. Let $s'_t$ be the chosen afterstate at time $t$.

\paragraph{TD Target.} For non-terminal transitions:
\[
y_t = \max_{a'} \left[ r_{t+1,a'} + \gamma \cdot V_{\theta^-}(s'_{t+1,a'}) \right]
\]
where $\theta^-$ is a target network updated every 200 episodes. For terminal transitions:
\[
y_t = \frac{p_{\text{invalid}}}{\gamma}
\]

\paragraph{Loss.} Huber loss (SmoothL1) with gradient clipping at norm 10.

\paragraph{Protocol.}
\begin{itemize}
    \item 10\,000 episodes, max 100 steps/episode.
    \item $\varepsilon$-greedy: $1.0\to0.01$, decay $\times0.999$/episode.
    \item Replay buffer: 100\,000 transitions, batch size 128.
    \item Adam, lr $=10^{-4}$, StepLR ($\gamma_{\text{sched}}=0.99$, step 1\,000).
    \item Checkpoints every 500 episodes with full training state; logging via Weights \& Biases.
\end{itemize}

\subsection{3P Planner}

For the 3P variant, a \texttt{RoundPlanner3P} wraps the DVN. At each new round it calls \texttt{get\_t\_plus\_3\_candidates($\gamma$)}, scores each candidate:
\[
\text{score} = R_{t+1:t+3} + \gamma^3 \cdot V_\theta(s_{t+3})
\]
and executes the best 3-action sequence, replanning if an action becomes invalid.

\section{Discussion and Future Work}

\paragraph{Why DVN outperforms DQN.}
The DVN decouples \emph{what the board looks like after the move} from \emph{which action achieves it}. The value network only needs to learn a board-to-value mapping---far simpler than the full Q-function which also encodes piece geometry and placement logic.

\paragraph{Limitations.}
\begin{itemize}
    \item The value function sees only the afterstate board, \emph{not} upcoming pieces.
    \item The 3P planner's full enumeration is $O(3!\times64^3)$ in the worst case---slow without batched GPU acceleration.
\end{itemize}

\paragraph{Future directions.}
\begin{itemize}
    \item Include next-piece information as extra input channels.
    \item Add a hand-crafted feature branch (holes, row/column fill density, height variance).
    \item Batch the 3P lookahead on GPU.
    \item Monte Carlo Tree Search guided by the value function.
\end{itemize}

%=============================================================================
\section{Conclusion}

We have presented a reinforcement learning study of Block Blast. Standard DQN approaches (DDQN, Rainbow, $\sim$2.67M parameters) struggled to converge within the training budget. A Deep Value Network operating on afterstates---with only 42\,461 parameters and a multi-kernel CNN architecture---achieved substantially better episode lengths and generalised to a three-piece planning setting. The key insight is that leveraging environment structure (precomputed afterstates) reduces the learning problem to simpler value estimation, making training significantly more sample-efficient.
